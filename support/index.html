What to include in a support request

  • Your macOS version (sw_vers) and SmrtDesk version (About → SmrtDesk).
  • Which provider(s) are misbehaving (Hue, Sonos, HomeKit, Home Assistant, etc.).
  • If a crash: the most recent entry in ~/Library/Application Support/SmrtDesk/diagnostics/events.jsonl or a screenshot of the in-app Diagnostics view.
  • Whether the issue happens on local-network operations or only cloud ones.

Common fixes

  • Hue bridge not discovered: macOS Local Network permission. System Settings → Privacy & Security → Local Network → enable SmrtDesk.
  • Cloud provider repeatedly asks to re-auth: Keychain entry may be stale. Quit SmrtDesk, open Keychain Access, search smrtdesk, delete the entry for that provider, relaunch and sign in again.
  • Alexa scene says "I can't find that": scene names take ~5 minutes to propagate after publishing. Re-link the SmrtDesk Skill in the Alexa app if it persists.
